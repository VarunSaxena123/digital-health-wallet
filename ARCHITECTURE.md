## ğŸ”‘ Key Design Decisions

### 1. JWT Authentication

**Why**: Stateless authentication, scalable, standard in industry

- Token stored in localStorage (accessible to JS)
- Included in Authorization header automatically
- 7-day expiration for security

### 2. SQLite Database

**Why**: Lightweight, serverless, perfect for single-instance apps

- No separate database server needed
- Easy to backup and transfer
- Sufficient for MVP and small deployments
- Can migrate to PostgreSQL later if needed

### 3. Local File Storage

**Why**: Simple initial implementation, full control

- Files stored in `uploads/` directory
- Logical filenames with timestamps to avoid collisions
- Can migrate to S3/Cloud Storage later
- Currently served as static files

### 4. Modular Component Architecture

**Why**: Reusability, maintainability, testability

- Separation of concerns
- Easy to unit test
- Simple to extend with new features
- Clear props interface for each component

### 5. Custom Hooks Pattern

**Why**: Logic reuse without class components

- useAuth: Global auth state management
- useForm: Reusable form handling logic
- useLoading: Loading state management
- Cleaner than prop drilling

### 6. Centralized API Service

**Why**: Single source of truth for API communication

- Consistent error handling
- Automatic token injection
- Easy to mock for testing
- Easy to change API URL per environment
